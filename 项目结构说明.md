# 项目结构说明

> 项目:智辉科技企业智能体(基于若依 RuoYi-ERP 二开)
> 更新:2026-07-16
> 标注说明:【核心】原版 ERP 功能,不能删 |【新增】二开加的 AI 获客/知识库功能 |【工具】辅助文件,可删

```
ruoyierp/
├── ruoyi-erp-vue/                  前端(Vite + Vue3 + Element Plus,端口 5173)
├── ruoyi-erp-service/              后端(Spring Boot 3 + Sa-Token + MyBatis,端口 8080)
├── deleted-menus-backup-20260716.txt  【工具】误删菜单的备份,确认无问题可删
├── restore-ai-menus.sql               【工具】菜单恢复脚本(历史),可删
└── 项目结构说明.md                     本文件
```

## 后端 ruoyi-erp-service/

```

│   │                           【核心】ERP 业务模块,不能删

│   ├── ruoyi-wechat/           【新增】企微对接(配置/客户拉取/AI托管/回调)

└── script/                     【核心】部署脚本
```

## 数据库(MySQL: erp,root/123456)

| 表 | 归属 | 说明 |
|---|---|---|
| sys_* / basic_* / wms_* / purchase_* / sales_* / financial_* | 【核心】 | ERP 原有表 |
| wechat_config / wechat_customer / wechat_hosting / wechat_message_log | 【新增】 | 企微模块 |
| rag_document / rag_chunk | 【新增】 | 知识库(rag_library 规划中) |

⚠ 本库字段语义与标准若依相反:`visible/status` 1=正常显示、`is_frame` 0=非外链。
⚠ MySQL 账号 `ai_readonly`:AI 模块专用只读账号(SqlGuard 配套),
   已额外授权 wechat_customer 读写(WechatCustomerController 误用此数据源,属技术债)。

## 三系统入口

| 系统 | 路由 | 说明 |
|---|---|---|
| ERP系统 | `/` | 原有 ERP,右上角下拉可切换系统 |
| AI获客系统 | `/ai-kit` | 客户导入/AI托管/企微配置/AI经营分析 |
| 知识库配置 | `/knowledge` | 知识库管理(MaxKB 风格改造中) |
